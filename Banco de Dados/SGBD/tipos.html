Relacional e Não Relacional

Banco de Dados Relacional

Um banco de dados relacional organiza os dados em tabelas que podem ser relacionadas entre si. Cada tabela contém linhas e colunas, onde as colunas representam os atributos dos dados e as linhas representam os registros. Exemplos populares de SGBDs relacionais incluem MySQL, PostgreSQL e Oracle Database.

Banco de Dados Não Relacional

Um banco de dados não relacional, também conhecido como NoSQL, armazena os dados de maneira diferente das tabelas tradicionais. Eles podem usar formatos como documentos, grafos, chave-valor ou colunas largas. Esses bancos de dados são frequentemente usados para grandes volumes de dados, alta escalabilidade e flexibilidade. Exemplos populares de SGBDs não relacionais incluem MongoDB, Cassandra e Redis.

Comparação entre Banco de Dados Relacional e Não Relacional

Característica Banco de Dados Relacional Banco de Dados Não Relacional
Estrutura Tabelas com linhas e colunas Documentos, grafos, chave-valor, colunas largas
Escalabilidade Vertical (aumentar capacidade do servidor) Horizontal (adicionar mais servidores)
Flexibilidade Menos flexível, esquema rígido Mais flexível, esquema dinâmico
Exemplos MySQL, PostgreSQL, Oracle Database MongoDB, Cassandra, Redis
